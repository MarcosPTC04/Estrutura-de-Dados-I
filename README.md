Este trabalho apresenta a resolução de cinco exercícios de lógica de programação desenvolvidos nas linguagens Java e Python, com o objetivo de praticar conceitos fundamentais como vetores/listas, estruturas de repetição, estruturas condicionais, operadores matemáticos e criação de funções (métodos). A proposta também permite comparar como cada linguagem resolve os mesmos problemas utilizando sintaxes diferentes, mas mantendo a mesma lógica.

No primeiro exercício, foi implementado um programa para processar um ranking de notas. Em ambas as linguagens, os valores foram armazenados em um vetor (em Java, double[]) ou em uma lista (em Python). O algoritmo percorre a estrutura utilizando um laço de repetição para somar todas as notas e calcular a média aritmética dividindo a soma pela quantidade de elementos. Em seguida, um novo laço conta quantos alunos obtiveram nota acima da média. Para encontrar a maior nota, o programa inicia uma variável com o primeiro elemento e compara com os demais, atualizando sempre que encontra um valor maior. A principal diferença entre as linguagens está na sintaxe: Java exige declaração explícita de tipos e uso de length, enquanto Python utiliza tipagem dinâmica e a função len().

No segundo exercício, foi desenvolvido um simulador de caixa eletrônico que calcula a menor quantidade possível de cédulas de R$ 50, R$ 20 e R$ 10 para um determinado valor inteiro múltiplo de 10. Tanto em Java quanto em Python, a solução utiliza divisão inteira e operador de resto para determinar quantas notas cabem no valor informado e quanto ainda resta após cada cálculo. Em Java, utiliza-se / com variáveis do tipo int, enquanto em Python utiliza-se o operador // para divisão inteira. Em ambas as linguagens, o algoritmo começa pelas notas de maior valor, garantindo o menor número possível de cédulas.

O terceiro exercício consiste na criação de um validador de senha segura. O programa verifica se a senha possui pelo menos oito caracteres e se contém pelo menos um caractere especial. Em Java, foi utilizado o método length() da classe String e o método charAt() para percorrer caractere por caractere. Em Python, utilizou-se a função len() e um laço for que percorre diretamente os caracteres da string. Em ambas as soluções, variáveis booleanas armazenam se cada critério foi atendido, e ao final o programa informa se a senha é válida ou inválida, indicando qual requisito não foi cumprido.

No quarto exercício, foi implementado um conversor de tempo que transforma um valor total em segundos para o formato horas, minutos e segundos. O cálculo é feito dividindo o total por 3600 para obter as horas, utilizando o operador de resto para descobrir o valor restante, depois dividindo por 60 para calcular os minutos e utilizando novamente o resto para encontrar os segundos finais. A lógica matemática é idêntica nas duas linguagens, variando apenas a sintaxe utilizada.

O quinto exercício envolve a criação de uma função que verifica se todos os elementos de um vetor ou lista são distintos. A solução implementada em ambas as linguagens utiliza dois laços de repetição aninhados: o primeiro percorre cada elemento e o segundo compara com os elementos subsequentes. Caso dois valores iguais sejam encontrados, o método ou função retorna que há números repetidos; caso contrário, informa que todos os elementos são únicos. Esse algoritmo possui complexidade O(n²), pois cada elemento é comparado com os demais.

Comparando as duas linguagens, observa-se que Java é uma linguagem estaticamente tipada, exigindo declaração explícita de tipos e maior estrutura no código, enquanto Python é dinamicamente tipada e apresenta uma sintaxe mais simples e enxuta. Apesar dessas diferenças, a lógica de resolução dos problemas permanece a mesma, demonstrando que a base da programação está no raciocínio lógico, independentemente da linguagem utilizada.

Em conclusão, a realização desses exercícios em Java e Python fortalece a compreensão dos fundamentos da programação estruturada, desenvolvendo habilidades de lógica, organização de código e análise de problemas. A comparação entre as linguagens evidencia que, embora a sintaxe mude, os princípios da programação são universais.
